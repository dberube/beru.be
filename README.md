# beru.be

Astro static site for David Berube — a proof-of-work hub for a commercial product leader who builds.

## Local commands

```bash
npm install
npm run dev
npm run build
npm run preview
```

The site is fully static (`output: 'static'`) and builds to `dist/` for any static server, including Caddy.

## Content model

All public content lives in Astro content collections. Adding or editing content should not require touching templates.

### Writing

Path: `src/content/writing/*.md`

Required frontmatter:

```yaml
title: "Essay title"
date: 2026-07-16
summary: "One or two direct sentences used for SEO and index pages."
draft: false
```

Optional:

```yaml
updated: 2026-07-17
tags: ["product", "agents"]
```

Draft essays are excluded from generated pages, indexes, RSS, and the homepage proof strip.

### Builds

Path: `src/content/builds/*.md`

Required frontmatter:

```yaml
name: "Project name"
summary: "One concrete sentence."
date: 2026-07-16
stack: ["Astro", "SQLite"]
status: live # live | archived | wip
draft: false
```

Optional:

```yaml
updated: 2026-07-17
url: "https://example.com"
repo: "https://github.com/user/repo"
screenshot: "../../assets/screenshots/example.png"
```

If the Markdown body has content, the build gets `/builds/[slug]/`. If the body is empty, list cards link to `url` when present.

### Agents

Path: `src/content/agents/*.md`

Same schema as builds, plus:

```yaml
runs_on: "Proxmox VPS"
```

Agents are systems working for David; builds are tools other people can use.

### Now

Current now page: `src/content/now/current.md`

Required frontmatter:

```yaml
updated: 2026-07-16
draft: false
```

Archive convention: before replacing `current.md`, manually copy the old file into `src/content/now/archive/YYYY-MM.md`. Archive entries render at `/now/YYYY-MM/` and are listed at the bottom of `/now/`.

## Homepage proof strip

The homepage is generated automatically. It merges live writing, builds, and agents; sorts by `updated ?? date` descending; and shows the top five. Do not edit the homepage to add items manually.

## SEO and feeds

- Canonicals, meta descriptions, OG tags, and Twitter cards are emitted by `BaseLayout`.
- Site URL is `https://beru.be` in `astro.config.mjs`.
- OG PNG cards are generated at build time into `public/og/` by `scripts/generate-og.mjs`.
- RSS feed is at `/feed.xml` and includes public writing entries only.
- Sitemap is generated by `@astrojs/sitemap`.

## Deployment

`deploy.sh` pulls `origin/main`, installs production dependencies, and builds `dist/`. It exits early when the remote commit matches local `HEAD`.

Cron example:

```cron
*/5 * * * * /home/hermes/development/beru.be/deploy.sh >> /var/log/berube-deploy.log 2>&1
```

Caddy example:

```caddy
beru.be, www.beru.be {
  root * /home/hermes/development/beru.be/dist
  file_server
}
```
